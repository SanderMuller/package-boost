# AI Guidelines & Skills Management

## Important: Never Edit Generated Files Directly

`CLAUDE.md`, `AGENTS.md`, `GEMINI.md`, and the per-agent skill dirs (`.claude/skills/`, `.cursor/skills/`, `.agents/skills/`, `.github/skills/`, `.junie/skills/`, `.kiro/skills/`) are **auto-generated** by `package-boost:sync`. Any direct edits will be overwritten on the next sync. **Always edit the source files in `.ai/` instead.**

## Project Structure

```
.ai/                              # SOURCE - Edit files here
├── guidelines/                   # Project-wide AI instructions
│   └── *.blade.php / *.md        # Coding conventions
└── skills/                       # Skill definitions
    └── {skill-name}/SKILL.md     # One folder per skill

.claude/skills/                   # AUTO-GENERATED — Claude Code
.cursor/skills/                   # AUTO-GENERATED — Cursor
.agents/skills/                   # AUTO-GENERATED — Codex / Gemini / OpenCode / Amp
.github/skills/                   # AUTO-GENERATED — GitHub Copilot
.junie/skills/                    # AUTO-GENERATED — Junie (JetBrains)
.kiro/skills/                     # AUTO-GENERATED — Kiro
└── {skill-name}/SKILL.md         # Skills for each agent
```

**CRITICAL**: Only edit files in `.ai/`. Any per-agent skill dir is auto-generated by `vendor/bin/testbench package-boost:sync` and will be overwritten. If you are about to edit a file under one of those dirs, STOP — find the corresponding file in `.ai/skills/` and edit that instead.

## How package-boost works (in Package Context)

This package uses package-boost via Orchestra Testbench:

1. **Guidelines** in `.ai/guidelines/` define project-wide coding conventions
2. **Skills** in `.ai/skills/` provide domain-specific instructions activated on demand
3. `package-boost:sync` composes these with package-boost's shipped foundation + any vendor-package contributions and writes per-agent guideline files (`CLAUDE.md`, `AGENTS.md`, `GEMINI.md`) and skill dirs (one per agent in your selection)
4. Run via `vendor/bin/testbench` instead of `php artisan`. Pick which agents to sync via `package-boost:install` (or set `agents` in `config/package-boost.php`); the default is all 9.

## Updating Guidelines

Edit files in `.ai/guidelines/`:

- **Blade files** (`.blade.php`): Use `@verbatim` blocks around code examples
- **Markdown files** (`.md`): Standard markdown format

## Creating/Updating Skills

Skills live in `.ai/skills/{skill-name}/SKILL.md` with frontmatter:

```markdown
---
name: skill-name
description: When this skill activates. Include trigger words and scenarios.
---

# Skill Title

## When to use this skill

- Scenario 1
- Scenario 2

## Instructions

Detailed instructions for the skill...
```

### Skill Naming

- Use kebab-case for directory names: `my-new-skill/`
- The `name` in frontmatter should match the directory name
- Write clear `description` with activation triggers

## Regenerating Generated Files

**CRITICAL: Always run `package-boost:sync` after editing any `.ai/` files.**

```bash
vendor/bin/testbench package-boost:sync
```

This syncs your `.ai/` changes to every selected agent's targets:

| Generated File / Directory | Purpose                                                |
|----------------------------|--------------------------------------------------------|
| `CLAUDE.md`                | Main context file for Claude Code                      |
| `AGENTS.md`                | Main context file for Cursor / Copilot / Codex / etc.  |
| `GEMINI.md`                | Main context file for Gemini CLI                       |
| `.claude/skills/`          | Skills for Claude Code                                 |
| `.cursor/skills/`          | Skills for Cursor                                      |
| `.agents/skills/`          | Skills for Codex / Gemini / OpenCode / Amp (shared)    |
| `.github/skills/`          | Skills for GitHub Copilot                              |
| `.junie/skills/`           | Skills for Junie (JetBrains)                           |
| `.kiro/skills/`            | Skills for Kiro                                        |

**All these files must be committed together** — they are kept in sync by `package-boost:sync`. To narrow this list to a subset of agents, run `package-boost:install` (writes the `agents` key in your workbench config).

## Checklist for Changes

- [ ] Edit files in `.ai/guidelines/` or `.ai/skills/` (never edit generated files)

- [ ] Include clear activation triggers in skill descriptions
- [ ] **Run `vendor/bin/testbench package-boost:sync`** (CRITICAL!)
- [ ] **Verify `git status`** shows changes in `.ai/` and the corresponding generated files
- [ ] **Commit ALL generated files together**
